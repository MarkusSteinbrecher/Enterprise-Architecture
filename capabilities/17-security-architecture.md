# Capability 17: Security Architecture

**Domain:** Technology Architecture

## Description

Define security controls, patterns, frameworks, and standards across the technology landscape. Security architecture ensures that the enterprise is protected against threats while enabling business agility.

## Objectives

- Define a comprehensive security architecture framework
- Ensure security-by-design across all architecture domains
- Protect data, applications, and infrastructure from threats
- Enable compliance with security regulations and standards

## Key Activities

- Security architecture framework definition
- Threat modelling and risk assessment
- Security control design and selection
- Identity and access management (IAM) architecture
- Zero trust architecture design
- Security pattern catalog development
- Security architecture review and assurance
- Incident response architecture
- Cloud security architecture

## Inputs

- Business and regulatory security requirements
- Threat intelligence and risk assessments
- Architecture blueprints (application, data, infrastructure)
- Security standards (ISO 27001, NIST, CIS)
- Compliance requirements
- Incident history and lessons learned

## Outputs / Deliverables

- Security architecture framework
- Security reference architecture
- Threat models
- Security control catalog
- IAM architecture
- Security standards and guidelines
- Security architecture review reports
- Security pattern catalog

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | Security addressed reactively; no architecture |
| Developing | Basic security controls defined; some standards in place |
| Defined | Security architecture framework established; security-by-design adopted |
| Managed | Security architecture actively enforced; continuous monitoring |
| Optimizing | Zero trust mature; automated security assurance; adaptive security |

## Key Roles

- Security Architect
- Chief Information Security Officer (CISO)
- Enterprise Architect
- Cloud Security Architect
- Identity Architect
- Security Engineer

## Tools & Techniques

- Threat modelling tools (e.g., STRIDE, MITRE ATT&CK)
- Security information and event management (SIEM)
- Identity and access management (IAM) platforms
- Security scanning and testing tools
- Zero trust frameworks
- Security architecture frameworks (SABSA, TOGAF)

## KPIs / Metrics

- Security incidents and breaches
- Security architecture review coverage
- Compliance audit results
- Mean time to detect/respond (MTTD/MTTR)
- Security control effectiveness
- Vulnerability remediation rate

## Relationships

- **Infrastructure Architecture** - Security controls at infrastructure layer
- **Application Design** - Security patterns in application design
- **Data Governance** - Data classification drives security controls
- **Compliance Management** - Security supports regulatory compliance
- **Risk Management** - Security risks feed into enterprise risk management
