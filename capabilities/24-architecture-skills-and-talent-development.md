# Capability 24: Architecture Skills & Talent Development

**Domain:** Architecture Management

## Purpose

To build, maintain, and continuously develop architecture skills, competencies, and career paths, ensuring the EA function has the talent needed to deliver on its mandate and adapt to evolving technology and business demands.

## Scope

- Architecture skill framework and competency model
- Skill gap assessment and development planning
- Training, certification, and mentoring programs
- Career path definition for architecture roles
- Community of practice management
- Recruitment and onboarding

## Key Activities

- Define architecture skill framework and competency model (technical, business, soft skills)
- Assess current skills and identify gaps against required competencies
- Develop and deliver training programs (internal and external)
- Support architecture certification (TOGAF, cloud certifications, domain certifications)
- Establish and run mentoring and coaching programs
- Define career paths and progression criteria for architecture roles
- Facilitate architecture communities of practice (CoPs)
- Organize knowledge sharing sessions (brown bags, tech talks, lunch-and-learns)
- Recruit and onboard new architecture talent
- Track and report on skills development progress

## Inputs

- Architecture role definitions and competency requirements
- Skill gap assessment results
- EA maturity assessment findings (Cap 23)
- Technology trends and emerging skill needs (Cap 25)
- Team feedback and career aspirations
- Industry skill frameworks (SFIA, TOGAF Skills Framework)

## Outputs / Deliverables

- Architecture skill framework / competency model
- Skill gap analysis reports
- Individual development plans
- Training and certification records
- Community of practice activity reports
- Career path documentation
- Mentoring program structure and materials
- Onboarding guides for new architects

## Roles & Responsibilities

| Role | Responsibility |
|------|---------------|
| Architecture Manager / Practice Lead | Owns skills strategy and development programs |
| Chief Architect | Sets direction for required competencies |
| Senior Architects | Serve as mentors and knowledge contributors |
| HR / Learning & Development | Supports training and career path frameworks |
| CoP Leads | Facilitate community activities and knowledge sharing |

## Maturity Levels

| Level | Name | Description |
|-------|------|-------------|
| 1 | Initial | No formal skill development; learning is ad-hoc and self-directed |
| 2 | Repeatable | Basic training budget available; some certification support; informal mentoring |
| 3 | Defined | Skill framework defined; structured development programs; CoP active; career paths documented |
| 4 | Managed | Skill gaps actively managed; development plans tracked; CoP thriving; certification rate measured |
| 5 | Optimizing | Learning culture embedded; architects as teachers; continuous competency evolution; talent pipeline strong |

## Key Metrics / KPIs

- Skill coverage rate (% of required skills present in team)
- Training hours per architect per year
- Certification achievement rate
- Community of practice participation rate
- Skill gap closure rate
- Employee satisfaction with development opportunities
- Architecture talent retention rate

## Dependencies

- **EA Practice & Operating Model** (Cap 22) - Skills built within the EA practice
- **EA Value Management** (Cap 23) - Maturity gaps inform skill needs
- **Innovation & Technology Radar** (Cap 25) - New technologies require new skills
- **Architecture Content Management** (Cap 20) - Repository used for learning and knowledge
- **Architecture Communication & Engagement** (Cap 26) - Communication skills are critical for architects

## Tools & Technologies

- Competency frameworks (SFIA, TOGAF Skills Framework, custom)
- Learning management systems (LMS)
- Certification platforms (TOGAF, AWS, Azure, GCP)
- Mentoring platforms
- Knowledge sharing platforms (Confluence, Notion, internal wikis)
- Conference and event management
