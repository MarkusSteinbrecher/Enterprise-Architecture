# Capability 13: Data Modelling

**Domain:** Data Architecture

## Description

Create conceptual, logical, and physical data models to represent data structures, relationships, and constraints. Data modelling provides a foundation for understanding, managing, and leveraging data as a strategic asset.

## Objectives

- Create accurate and comprehensive data models at all levels of abstraction
- Establish a common data vocabulary across the enterprise
- Support database design and data integration efforts
- Enable data quality and governance through clear data definitions

## Key Activities

- Conceptual data modelling (business entities and relationships)
- Logical data modelling (detailed attributes, keys, normalization)
- Physical data modelling (database-specific design)
- Data dictionary and glossary maintenance
- Entity-relationship modelling
- Dimensional modelling (for analytics)
- Data model versioning and governance
- Data model review and validation

## Inputs

- Business requirements and processes
- Existing data stores and schemas
- Data standards and naming conventions
- Industry data models and standards
- Regulatory data requirements

## Outputs / Deliverables

- Conceptual data models
- Logical data models
- Physical data models
- Data dictionary / glossary
- Entity-relationship diagrams
- Dimensional models (star/snowflake schemas)
- Data model documentation

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | No formal data models; data structures ad-hoc |
| Developing | Key data entities modelled; basic data dictionary |
| Defined | Enterprise data model maintained; standard modelling practices |
| Managed | Data models actively govern schema changes; comprehensive dictionary |
| Optimizing | Automated data model generation; models drive data quality |

## Key Roles

- Data Architect
- Data Modeller
- Database Administrator
- Enterprise Architect
- Business Data Steward

## Tools & Techniques

- Data modelling tools (e.g., ERwin, PowerDesigner, dbdiagram.io)
- ER modelling notation
- Dimensional modelling techniques (Kimball, Inmon)
- Data standards and naming conventions
- Model review checklists

## KPIs / Metrics

- Data model coverage (% of key entities modelled)
- Data dictionary completeness
- Model currency (last update date)
- Number of data model violations detected
- Data model reuse rate

## Relationships

- **Data Governance** - Data models support governance policies
- **Data Integration & Analytics** - Models inform integration and analytics design
- **Application Design** - Data models influence application data layers
- **Standards & Reference Architectures** - Data standards inform modelling
- **Architecture Knowledge Management** - Data models are key knowledge assets
