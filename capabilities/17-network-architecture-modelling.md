# Capability 17: Network Architecture Modelling

**Domain:** Architecture Development – Technology Architecture

**Tagline:** Design and optimize the network infrastructure that enables secure and reliable connectivity.

## Definition

Network Architecture Modelling involves designing and documenting the network infrastructure and connectivity that enables communication between systems, applications, users, and external partners. It defines the network topology and its components.

## Objectives

Design robust and secure network architectures that support business connectivity requirements while optimizing performance, availability, security, and cost. Ensure consistent network standards and patterns across the enterprise. Enable effective network planning, security segmentation, and operational management.

## Tasks

1. **Identify network requirements** from application architecture, business requirements, and non-functional requirements.
2. **Design network topology,** segmentation, security zones, based on security and compliance requirements.
3. **Define connectivity, capacity and performance** for different scenarios (site-to-site, remote access, cloud connectivity, partner integration).
4. **Apply network architecture patterns and standards** (e.g., redundancy, high availability, traffic management, Quality of Service).
5. **Validate network architecture** with stakeholders (security architects, infrastructure teams, application architects). Document security and compliance requirements.
6. **Maintain and evolve** network architecture as technology and business requirements change.

## Inputs

- Application architecture and connectivity requirements
- Non-functional requirements
- Technology standards and reference architectures
- Existing network infrastructure, topology and usage data.
- Security and compliance policies, cost constraints

## Outputs

- Network architecture diagrams and topology models
- Network component specifications and configurations
- Network segmentation and security zone design
- High availability, capacity and performance design.
- Security and compliance requirements

## Roles

- Technology Architect (A)
- Security Architect (R)
- Application Architect (C)
- Lead Enterprise Architect (I)

## Common Issues with an Immature Capability

- Inconsistent network designs across environments, creating complexity.
- Performance bottlenecks due to inadequate capacity planning or poor traffic management.
- Single points of failure and inadequate redundancy planning.
- Complex firewall rules that are difficult to manage and audit.
- Poor visibility into network traffic and security events.
- High costs from over-provisioning or inefficient network design.

## Pragmatic Approach – Essentials That Are Needed for Success

- Start with security requirements - segmentation and security zones first.
- Use reference architectures and proven patterns.
- Design for zero-trust principles – and segment based on risk.
- Standardize on platforms and patterns - reduce complexity and ops overhead.
- Plan for redundancy and failover - eliminate single points of failures.
- Automate network management and Build in monitoring and observability.
- Document clearly - include rationale for design decisions.
