# Capability 6: Roadmap Development

**Domain:** Architecture Development - Common

## Description

Create transition plans and roadmaps for moving from the current to target architecture states. This includes defining work packages, sequencing, dependencies, and milestones for architecture transformation.

## Objectives

- Define a clear and achievable path from current to target state
- Sequence initiatives to maximize value and manage risk
- Identify dependencies and critical path items
- Enable portfolio and investment planning

## Key Activities

- Gap analysis between current and target states
- Work package definition and scoping
- Initiative sequencing and dependency mapping
- Transition architecture definition (intermediate states)
- Resource and capability planning
- Milestone and checkpoint definition
- Roadmap visualization and communication
- Roadmap review and adaptation

## Inputs

- Current state assessment
- Target state architecture
- Gap analysis results
- Business priorities and constraints
- Resource availability and budgets
- Program and project plans

## Outputs / Deliverables

- Architecture roadmap
- Transition architectures
- Work package definitions
- Implementation and migration plans
- Dependency maps
- Investment / business case inputs
- Milestone and checkpoint schedules

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | No roadmap; changes happen opportunistically |
| Developing | Basic roadmap exists but not actively maintained |
| Defined | Comprehensive roadmap linked to strategy; regularly reviewed |
| Managed | Roadmap drives portfolio decisions; actively tracked and updated |
| Optimizing | Adaptive roadmapping; continuous re-prioritization based on value delivery |

## Key Roles

- Enterprise Architect
- Architecture Manager
- Portfolio Manager
- Program Manager
- Business planners

## Tools & Techniques

- Roadmapping tools and templates
- Portfolio management tools
- Dependency mapping techniques
- Value stream mapping
- Agile release planning
- Cost-benefit analysis

## KPIs / Metrics

- Roadmap coverage of target state gaps
- Percentage of roadmap items on track
- Roadmap update frequency
- Stakeholder confidence in roadmap
- Value delivered against roadmap milestones

## Relationships

- **Current State Assessment** - Provides the starting point
- **Target State Design** - Provides the destination
- **Architecture Review & Approval** - Roadmap reviewed by governance
- **Change Management** - Roadmap changes managed through change control
- **Architecture Communication** - Roadmap is a key communication artifact
