# Capability 14: Data Governance

**Domain:** Data Architecture

## Description

Establish policies, standards, and processes for data quality, security, privacy, and lifecycle management. Data governance ensures that data is treated as a valuable enterprise asset with clear ownership and accountability.

## Objectives

- Ensure data quality, consistency, and reliability across the enterprise
- Establish clear data ownership and stewardship
- Comply with data privacy and regulatory requirements (GDPR, etc.)
- Define and enforce data policies and standards

## Key Activities

- Data governance framework definition
- Data ownership and stewardship assignment
- Data quality management (profiling, cleansing, monitoring)
- Data classification and sensitivity labeling
- Data privacy and protection policy definition
- Data lifecycle management (creation, use, archival, deletion)
- Master data management
- Data lineage and provenance tracking
- Data governance board facilitation

## Inputs

- Regulatory and compliance requirements (GDPR, CCPA, etc.)
- Business data requirements
- Data quality assessments
- Data architecture and models
- Industry data governance frameworks

## Outputs / Deliverables

- Data governance framework and charter
- Data policies and standards
- Data ownership / stewardship matrix (RACI)
- Data quality reports and dashboards
- Data classification scheme
- Data lifecycle policies
- Master data management strategy
- Data lineage documentation

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | No formal data governance; data quality is inconsistent |
| Developing | Basic data policies exist; data ownership partially assigned |
| Defined | Formal governance framework; data stewards active; quality measured |
| Managed | Data governance board operational; policies enforced; quality improving |
| Optimizing | Automated data quality; proactive governance; data-driven culture |

## Key Roles

- Data Governance Lead / Chief Data Officer
- Data Stewards
- Data Architect
- Data Quality Analyst
- Privacy Officer / DPO
- Enterprise Architect

## Tools & Techniques

- Data governance platforms (e.g., Collibra, Alation, Informatica)
- Data quality tools
- Data catalog tools
- Master data management (MDM) platforms
- Data lineage tools
- Privacy impact assessment frameworks

## KPIs / Metrics

- Data quality scores (accuracy, completeness, timeliness, consistency)
- Percentage of data assets with assigned ownership
- Data policy compliance rate
- Number of data-related incidents
- Regulatory compliance audit results
- Data catalog adoption rate

## Relationships

- **Data Modelling** - Governance policies applied through data models
- **Data Integration & Analytics** - Governance ensures quality data for analytics
- **Compliance Management** - Data governance supports regulatory compliance
- **Security Architecture** - Data classification informs security controls
- **Risk Management** - Data risks managed through governance
