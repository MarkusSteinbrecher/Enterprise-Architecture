# Capability 15: Data Integration & Analytics

**Domain:** Data Architecture

## Description

Design data flows, integration patterns, and analytics architectures to enable data-driven decision making. This capability covers how data moves across the enterprise and how it is transformed into actionable insights.

## Objectives

- Enable seamless data flow across systems and domains
- Design scalable analytics and reporting architectures
- Support self-service analytics and data democratization
- Ensure data is accessible, timely, and fit for purpose

## Key Activities

- Data integration architecture design (ETL/ELT, streaming, batch)
- Data pipeline design and management
- Data warehouse and data lake architecture
- Analytics platform architecture (BI, advanced analytics, AI/ML)
- Data virtualization and federation design
- Real-time data streaming architecture
- Self-service analytics enablement
- Data product design

## Inputs

- Business intelligence and analytics requirements
- Data models and data catalog
- Data governance policies
- Technology standards and platforms
- Data source inventory
- Performance and scalability requirements

## Outputs / Deliverables

- Data integration architecture
- Data pipeline designs
- Data warehouse / data lake architecture
- Analytics platform architecture
- Data flow diagrams
- Data product specifications
- Analytics and reporting frameworks

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | Manual data extracts; siloed reporting |
| Developing | Basic ETL processes; centralized data warehouse |
| Defined | Modern data platform; standard integration patterns; BI tools deployed |
| Managed | Self-service analytics; data products defined; quality monitored |
| Optimizing | Real-time analytics; AI/ML integrated; data mesh / data products |

## Key Roles

- Data Architect
- Data Engineer
- Analytics / BI Architect
- Enterprise Architect
- Data Scientist
- Analytics Product Owner

## Tools & Techniques

- ETL/ELT tools (e.g., dbt, Informatica, Talend)
- Data warehouse/lake platforms (e.g., Snowflake, Databricks, BigQuery)
- Streaming platforms (e.g., Kafka, Flink)
- BI tools (e.g., Tableau, Power BI, Looker)
- Data catalog and lineage tools
- Data mesh and data product frameworks

## KPIs / Metrics

- Data pipeline reliability and latency
- Analytics platform adoption rate
- Self-service analytics usage
- Data freshness (time to availability)
- Number of data products published
- Analytics-driven decision rate

## Relationships

- **Data Modelling** - Data models inform integration design
- **Data Governance** - Governance ensures quality in analytics
- **Integration Architecture** - Data integration aligned with application integration
- **Platform Architecture** - Analytics platforms as shared services
- **Innovation & Emerging Technology** - AI/ML and emerging analytics capabilities
