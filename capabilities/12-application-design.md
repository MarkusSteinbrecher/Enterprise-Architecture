# Capability 12: Application Design

**Domain:** Application Architecture

## Description

Define application structure, components, design patterns, and technology choices for individual applications or application groups. This capability ensures applications are well-architected, maintainable, and aligned with enterprise standards.

## Objectives

- Define consistent application design principles and patterns
- Ensure applications are scalable, maintainable, and secure
- Align application design with enterprise architecture standards
- Enable reuse of components and services

## Key Activities

- Application architecture definition (layers, components, services)
- Design pattern selection and application
- Technology stack selection
- Non-functional requirements realization (performance, scalability, security)
- Microservices and modular design
- Application security architecture
- DevOps and deployment architecture design
- Application design review

## Inputs

- Architecture requirements (functional and non-functional)
- Enterprise architecture standards and principles
- Technology standards and roadmaps
- Integration architecture patterns
- Security requirements and frameworks
- Application portfolio context

## Outputs / Deliverables

- Application architecture documents
- Component and service designs
- Technology stack decisions
- Design pattern specifications
- Deployment architecture diagrams
- Application security design
- Architecture decision records (ADRs)

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | Ad-hoc application design; no standards or patterns |
| Developing | Basic design patterns adopted; some standardization |
| Defined | Standard design patterns and technology stacks; design reviews in place |
| Managed | Consistent application design; reuse and component libraries active |
| Optimizing | Self-service application templates; automated design validation |

## Key Roles

- Application Architect
- Solution Architect
- Tech Lead / Lead Developer
- Enterprise Architect
- Security Architect

## Tools & Techniques

- Architecture description languages (C4 model, UML)
- Design pattern catalogs
- Application architecture frameworks
- Code analysis and quality tools
- Cloud-native design patterns
- Domain-driven design (DDD)

## KPIs / Metrics

- Architecture review pass rate
- Technical debt in applications
- Component reuse rate
- Standard compliance score
- Application deployment frequency and reliability

## Relationships

- **Application Portfolio Management** - Application designs contribute to portfolio
- **Integration Architecture** - Application interfaces designed for integration
- **Security Architecture** - Security patterns embedded in design
- **Platform Architecture** - Applications designed for target platforms
- **Standards & Reference Architectures** - Standards guide application design
