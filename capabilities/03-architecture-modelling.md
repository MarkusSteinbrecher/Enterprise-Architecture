# Capability 3: Architecture Modelling

**Domain:** Architecture Development - Common

## Description

Create, maintain, and communicate architecture models using standard notations, frameworks, and viewpoints. This capability enables consistent representation and analysis of architecture across domains.

## Objectives

- Provide clear and consistent representations of the architecture
- Enable analysis and communication through models
- Ensure models are maintained and kept current
- Support decision-making through visual and analytical models

## Key Activities

- Architecture model creation using standard notations (e.g., ArchiMate, UML, BPMN)
- Viewpoint and view definition for different stakeholder audiences
- Model repository management and versioning
- Model validation and consistency checking
- Model-based analysis (impact, dependency, gap)
- Model publishing and distribution

## Inputs

- Architecture requirements
- Current and target state information
- Architecture frameworks and metamodels
- Stakeholder viewpoint definitions
- Existing models and documentation

## Outputs / Deliverables

- Architecture models (current state, target state, transition)
- Architecture views tailored to stakeholder viewpoints
- Model catalogs, matrices, and diagrams
- Gap analysis reports
- Model quality reports

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | Ad-hoc diagrams; no standard notation or tooling |
| Developing | Some standard notation used; models created per project |
| Defined | Consistent use of modelling standards; central model repository |
| Managed | Models actively maintained; used for analysis and decision-making |
| Optimizing | Automated model generation and validation; models drive change |

## Key Roles

- Enterprise Architect
- Solution Architect
- Domain Architect (Business, Application, Data, Technology)
- Architecture Modeller

## Tools & Techniques

- Architecture modelling tools (e.g., Archi, Sparx EA, LeanIX, Mega)
- Standard notations (ArchiMate, UML, BPMN, C4)
- Viewpoint frameworks (TOGAF, Zachman)
- Model-based analysis techniques

## KPIs / Metrics

- Number of models maintained and current
- Model coverage across architecture domains
- Model reuse rate
- Stakeholder model satisfaction
- Model update frequency

## Relationships

- **Current State Assessment** - Modelling captures baseline architecture
- **Target State Design** - Modelling represents future state
- **Architecture Knowledge Management** - Models are key architecture artifacts
- **Architecture Tooling** - Modelling depends on appropriate tools
- **Architecture Communication** - Models are primary communication vehicles
