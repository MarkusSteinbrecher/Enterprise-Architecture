# Capability 21: Repository Support & Maintenance

**Domain:** Architecture Repository

**Tagline:** Keep the architecture repository operational, reliable and fit for evolving organizational needs.

## Definition

Managing the day-to-day operations, technical maintenance and continuous improvement of the architecture repository platform, ensuring its availability, usability and alignment with evolving EA requirements.

## Objectives

Ensure the architecture repository is stable, performant and available for all EA stakeholders. Maintain the repository structure, tooling and integrations as organizational needs evolve. Provide user support and enablement to drive adoption and effective usage of the repository.

## Tasks

1. **Manage the EA repository platform** (e.g. LeanIX, Mega, ServiceNow) including configuration, access management and upgrades.
2. **Maintain and evolve the repository meta-model** and data structure as EA requirements change.
3. **Manage integrations** with connected systems (CMDB, ITSM, PPM, etc.) and ensure data flows are operational.
4. **Provide user support** – handle requests, issues and questions from architects and stakeholders.
5. **Develop and maintain** user guidelines, templates and onboarding materials.
6. **Monitor platform usage,** performance and adoption metrics; identify and implement improvements.

## Inputs

- EA repository platform and infrastructure
- User feedback and support requests
- EA meta-model and data model requirements
- Integration specifications and SLAs with connected systems

## Outputs

- Operational and up-to-date repository platform
- Updated meta-model and repository configuration
- User guidelines, templates and training materials
- Platform usage and adoption reports, resolved support tickets

## Roles

- EA Repository Manager / EA Tool Administrator (R)
- Lead Enterprise Architect (A)
- IT Operations / Platform Support (C)
- Domain Architects & EA Users (I)

## Common Issues with an Immature Capability

- The repository tool is set up but nobody maintains it – meta-model drifts, integrations break, data becomes stale.
- Low adoption because users find the tool complex, unclear or poorly documented.
- No dedicated ownership – repository maintenance is "everyone's and no one's job."

## Pragmatic Approach – Essentials That Are Needed for Success

- Assign a dedicated repository owner with clear accountability.
- Keep the meta-model simple and evolve incrementally – avoid over-modelling upfront.
- Invest in user onboarding and short how-to guides – adoption depends on usability.
- Monitor usage regularly – if nobody uses it, find out why and fix it before adding more features.
