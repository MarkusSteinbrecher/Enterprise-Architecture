# Capability 30: Change Management

**Domain:** Governance

## Description

Manage architecture changes and their impact across the enterprise. This capability ensures that changes to the architecture are controlled, communicated, and aligned with the overall architecture direction.

## Objectives

- Ensure architecture changes are controlled and deliberate
- Assess the impact of changes across architecture domains
- Communicate changes to affected stakeholders
- Maintain architecture integrity through change

## Key Activities

- Change request management for architecture
- Impact assessment across architecture domains
- Change approval and prioritization
- Architecture change communication
- Change implementation tracking
- Post-implementation review
- Change calendar and release coordination
- Emergency change procedures
- Architecture versioning and baselining

## Inputs

- Architecture change requests
- Business and technology change drivers
- Architecture baselines and current state
- Stakeholder impact analysis
- Risk assessments
- Roadmap and transition plans

## Outputs / Deliverables

- Change request log
- Impact assessment reports
- Change approval decisions
- Change communication plans
- Updated architecture baselines
- Post-implementation review reports
- Change metrics and reports
- Architecture version history

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | Architecture changes uncontrolled; no formal process |
| Developing | Basic change tracking; some impact assessment |
| Defined | Formal change management process; impact assessment standard |
| Managed | Change governance active; cross-domain impact analysis; metrics tracked |
| Optimizing | Automated impact analysis; change velocity optimized; continuous delivery |

## Key Roles

- Enterprise Architect
- Architecture Manager
- Change Manager / CAB (Change Advisory Board)
- Solution Architect
- Stakeholder representatives
- Release Manager

## Tools & Techniques

- Change management tools (ServiceNow, Jira)
- Impact analysis techniques and tools
- Architecture modelling tools (for impact visualization)
- Communication platforms
- Change Advisory Board (CAB) processes
- ITIL change management framework

## KPIs / Metrics

- Number of architecture changes per period
- Change approval cycle time
- Change success rate (implemented without issues)
- Emergency change rate (lower is better)
- Stakeholder satisfaction with change process
- Architecture drift from baseline

## Relationships

- **Roadmap Development** - Roadmap changes managed through change control
- **Risk Management** - Changes assessed for risk impact
- **Architecture Review & Approval** - Significant changes reviewed by governance
- **Architecture Communication** - Changes communicated to stakeholders
- **Current State Assessment** - Baseline updated after changes
