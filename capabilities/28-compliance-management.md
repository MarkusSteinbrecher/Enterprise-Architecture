# Capability 28: Compliance Management

**Domain:** Governance

## Description

Ensure architecture compliance with internal standards, external regulations, and organizational policies. This capability tracks, measures, and reports on how well the enterprise adheres to defined architecture guardrails.

## Objectives

- Ensure adherence to architecture standards and principles
- Track and report compliance across the portfolio
- Manage non-compliance and exceptions effectively
- Support regulatory compliance through architecture

## Key Activities

- Compliance framework definition
- Compliance assessment and auditing
- Non-compliance identification and remediation tracking
- Exception and waiver management
- Regulatory compliance mapping (architecture controls to regulations)
- Compliance reporting and dashboarding
- Compliance automation and tooling
- Compliance process improvement

## Inputs

- Architecture standards and principles
- Regulatory and legal requirements
- Architecture review findings
- Portfolio assessment data
- Audit results
- Exception requests

## Outputs / Deliverables

- Compliance assessment reports
- Compliance dashboards and scorecards
- Non-compliance register and remediation plans
- Exception / waiver register
- Regulatory compliance mapping
- Compliance trend reports
- Audit evidence and documentation

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | No compliance tracking; standards exist but are not enforced |
| Developing | Basic compliance checking during reviews; limited tracking |
| Defined | Formal compliance framework; regular assessments; reporting established |
| Managed | Active compliance monitoring; exception management; trend analysis |
| Optimizing | Automated compliance checking; proactive compliance; continuous assurance |

## Key Roles

- Enterprise Architect
- Compliance Officer / Manager
- Architecture Review Board
- Internal Audit
- Security and Risk teams

## Tools & Techniques

- Compliance management platforms
- Architecture fitness functions (automated checks)
- GRC (Governance, Risk, Compliance) tools
- Portfolio analysis tools
- Audit frameworks and checklists
- Compliance dashboarding tools

## KPIs / Metrics

- Overall compliance rate
- Number of non-compliance items and severity
- Remediation time for non-compliance
- Number of exceptions / waivers active
- Audit findings related to architecture
- Compliance trend over time

## Relationships

- **Standards & Reference Architectures** - Standards define what compliance means
- **Architecture Review & Approval** - Reviews identify compliance gaps
- **Risk Management** - Non-compliance creates risk
- **Data Governance** - Data compliance is a key area
- **Security Architecture** - Security compliance requirements
