# Capability 10: Application Portfolio Management

**Domain:** Application Architecture

## Purpose

To maintain comprehensive oversight of the application landscape, manage application lifecycles, drive rationalization, and optimize the portfolio for cost, risk, and strategic alignment.

## Scope

- Application inventory and catalog management
- Application health and fitness assessment
- Application lifecycle management (plan, build, run, retire)
- Portfolio rationalization and consolidation
- Application-to-capability and application-to-technology mapping

## Key Activities

- Maintain a complete and accurate application inventory
- Assess applications on multiple dimensions (technical fitness, business value, cost, risk)
- Classify applications using frameworks (e.g., TIME: Tolerate, Invest, Migrate, Eliminate)
- Manage application lifecycles from introduction to retirement
- Plan and execute portfolio rationalization (reduce duplication, consolidate)
- Map applications to business capabilities, value streams, and technology platforms
- Calculate total cost of ownership (TCO) for applications
- Identify and manage shadow IT
- Plan application roadmaps (investment, migration, retirement)

## Inputs

- Application inventory data (from CMDB, discovery tools)
- Business capability model (Cap 7)
- Technology standards and lifecycle information (Cap 6)
- Business strategy and priorities
- Cost, performance, and usage data
- User feedback and satisfaction data

## Outputs / Deliverables

- Application portfolio catalog
- Application assessment scorecards
- TIME classification / heat maps
- Rationalization recommendations and plans
- Application lifecycle roadmaps
- TCO reports
- Application-to-capability heat maps
- Shadow IT register

## Roles & Responsibilities

| Role | Responsibility |
|------|---------------|
| Application Architect | Leads portfolio analysis and rationalization |
| Enterprise Architect | Ensures portfolio alignment with enterprise architecture |
| Application Owners | Provide application-level input and own lifecycle decisions |
| IT Asset Manager | Supports inventory and cost data |

## Maturity Levels

| Level | Name | Description |
|-------|------|-------------|
| 1 | Initial | No centralized application inventory; shadow IT prevalent and unmanaged |
| 2 | Repeatable | Basic application catalog maintained; limited assessment criteria |
| 3 | Defined | Comprehensive portfolio with regular assessments; TIME classification applied; rationalization planned |
| 4 | Managed | Portfolio actively governed; investment decisions driven by portfolio data; TCO tracked |
| 5 | Optimizing | Continuous portfolio optimization; automated discovery and assessment; portfolio drives strategy execution |

## Key Metrics / KPIs

- Total application count and growth/shrink rate
- Application health score distribution
- Redundancy ratio (duplicate capability coverage)
- Portfolio rationalization savings achieved
- Application retirement rate
- Shadow IT discovery rate
- TCO per application category

## Dependencies

- **Business Capability Modelling** (Cap 7) - Applications mapped to business capabilities
- **Application Integration & API Management** (Cap 11) - Integration complexity across portfolio
- **Infrastructure & Cloud Architecture** (Cap 16) - Applications deployed on infrastructure
- **Current State Assessment** (Cap 3) - Application portfolio is part of baseline
- **Gap Analysis & Roadmapping** (Cap 5) - Portfolio decisions feed the roadmap
- **Solution Design & Patterns** (Cap 12) - New applications designed using standard patterns

## Tools & Technologies
