# Capability 26: Architecture Communication & Engagement

**Domain:** Architecture Management

## Purpose

To effectively communicate architecture vision, decisions, standards, roadmaps, and value to diverse audiences — from executives to developers — ensuring architecture guidance is understood, adopted, and drives action.

## Scope

- Architecture communication strategy and planning
- Audience-specific communication delivery
- Architecture visualization and storytelling
- Architecture awareness and adoption campaigns
- Feedback collection and communication improvement

## Key Activities

- Develop architecture communication strategy (audiences, channels, cadence, key messages)
- Create audience-tailored architecture presentations and materials
- Deliver executive architecture briefings
- Publish architecture newsletters, blogs, or bulletins
- Conduct architecture brown bags, lunch-and-learns, and workshops
- Create architecture visualizations, infographics, and dashboards
- Tell the architecture story (connecting architecture to business outcomes)
- Run architecture awareness campaigns for new standards, patterns, or guidelines
- Collect feedback on communication effectiveness and iterate
- Maintain architecture intranet / portal content
- Present at internal tech conferences and events

## Inputs

- Architecture deliverables, decisions, and roadmaps
- Stakeholder analysis and audience profiles (Cap 1)
- EA value reports (Cap 23)
- Architecture principles and standards (Cap 6)
- Organizational communication channels and preferences

## Outputs / Deliverables

- Architecture communication plan
- Executive briefing materials
- Architecture presentations (tailored by audience)
- Architecture newsletters / bulletins
- Visualizations, infographics, and dashboards
- Architecture portal / intranet content
- Communication effectiveness reports

## Roles & Responsibilities

| Role | Responsibility |
|------|---------------|
| Chief Architect | Owns executive-level architecture narrative |
| Architecture Manager | Manages communication strategy and planning |
| Enterprise Architects | Create and deliver architecture presentations |
| All Architects | Serve as architecture ambassadors in their engagements |
| Communications / Marketing support | Supports professional content creation |

## Maturity Levels

| Level | Name | Description |
|-------|------|-------------|
| 1 | Initial | Architecture communicated informally; limited awareness outside EA team |
| 2 | Repeatable | Some architecture presentations shared; ad-hoc communication |
| 3 | Defined | Formal communication plan; regular cadence; multiple channels used; architecture portal live |
| 4 | Managed | Tailored communication by audience; feedback loops active; high awareness measured |
| 5 | Optimizing | Architecture brand established; high engagement and adoption; architecture communication drives behavior change |

## Key Metrics / KPIs

- Architecture awareness level (measured by survey)
- Communication reach (views, attendance at events)
- Stakeholder satisfaction with architecture communication
- Architecture guidance adoption rate
- Newsletter / portal engagement metrics
- Number of communication activities delivered per period

## Dependencies

- **Stakeholder Management** (Cap 1) - Communication tailored to stakeholder needs
- **Architecture Content Management** (Cap 20) - Repository supports communication content
- **EA Practice & Operating Model** (Cap 22) - Communication demonstrates practice value
- **EA Value Management** (Cap 23) - Value findings communicated to stakeholders
- **Gap Analysis & Roadmapping** (Cap 5) - Roadmap is a key communication artifact
