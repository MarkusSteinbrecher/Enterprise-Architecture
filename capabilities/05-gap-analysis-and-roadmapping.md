# Capability 5: Gap Analysis & Roadmapping

**Domain:** Architecture Development - Common

## Purpose

To systematically identify gaps between the current and target architecture states, and to create actionable transition plans and roadmaps that define the path, sequencing, dependencies, and milestones for architecture transformation.

## Scope

- Gap identification and analysis across architecture domains
- Work package definition and sequencing
- Transition architecture planning
- Roadmap creation and maintenance
- Dependency and critical path management

## Key Activities

- Perform gap analysis between current and target states across all domains
- Identify and classify gaps (capability gaps, technology gaps, process gaps, skill gaps)
- Define work packages and initiatives to close gaps
- Sequence initiatives based on priority, dependencies, and value
- Define transition architectures (intermediate states)
- Create architecture roadmaps with milestones and checkpoints
- Map dependencies between initiatives
- Align architecture roadmap with business and IT portfolio plans
- Develop business cases and investment justifications for roadmap items
- Review and adapt the roadmap periodically

## Inputs

- Current state assessment and baseline
- Target state architecture and blueprints
- Business priorities and constraints
- Resource availability and budgets
- Program and project plans
- Risk assessments

## Outputs / Deliverables

- Gap analysis report (current vs. target, by domain)
- Architecture roadmap
- Transition architecture definitions
- Work package descriptions
- Implementation and migration plans
- Dependency maps
- Investment / business case inputs
- Milestone and checkpoint schedule

## Roles & Responsibilities

| Role | Responsibility |
|------|---------------|
| Enterprise Architect | Leads gap analysis and roadmap development |
| Architecture Manager | Coordinates roadmap alignment with portfolio |
| Domain Architects | Contribute domain-specific gap analysis |
| Portfolio / Program Manager | Aligns roadmap with delivery capacity |

## Maturity Levels

| Level | Name | Description |
|-------|------|-------------|
| 1 | Initial | No formal gap analysis or roadmap; changes happen opportunistically |
| 2 | Repeatable | Basic gap analysis done per project; high-level roadmap exists |
| 3 | Defined | Comprehensive gap analysis across domains; roadmap linked to strategy and regularly reviewed |
| 4 | Managed | Roadmap actively drives portfolio decisions; progress tracked against milestones; dependencies managed |
| 5 | Optimizing | Adaptive roadmapping with continuous re-prioritization; value-driven sequencing; automated impact analysis |

## Key Metrics / KPIs

- Roadmap coverage (% of target state gaps with planned initiatives)
- Percentage of roadmap items on track
- Gap closure rate per period
- Roadmap update frequency
- Stakeholder confidence in roadmap (survey)
- Business case approval rate

## Dependencies

- **Current State Assessment** (Cap 3) - Provides the starting point for gap analysis
- **Target State Design** (Cap 4) - Provides the destination
- **Architecture Decision Management** (Cap 27) - Roadmap decisions governed
- **Change & Configuration Management** (Cap 30) - Roadmap changes managed through change control
- **Architecture Communication & Engagement** (Cap 26) - Roadmap is a key communication artifact
- **EA Value Management** (Cap 23) - Roadmap progress contributes to EA value reporting
