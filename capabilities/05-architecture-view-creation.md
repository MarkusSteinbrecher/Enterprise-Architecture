# Capability 5: Architecture View Creation

**Domain:** Architecture Development – Common

**Tagline:** Facilitate effective decision-making by addressing specific stakeholder concerns.

## Definition

Architecture View Creation involves developing various perspectives (views) of the architecture to address the concerns of different stakeholders. These views help in understanding, analysing, and communicating the architecture.

## Objectives

Provide tailored views that translate complex architecture data into clear, stakeholder-relevant perspectives. Enable informed decision-making by ensuring views are accurate, up-to-date and aligned with stakeholder concerns.

## Tasks

1. **Identify information needs** of EA stakeholders and their key questions/concerns.
2. **Select viewpoints** and define which views address which stakeholder concerns (e.g. business capability map, application landscape, technology lifecycle).
3. **Develop architecture views** using data from the architecture repository.
4. **Validate views** with stakeholders and iterate based on feedback.
5. **Maintain and update views** as architecture data evolves.

## Inputs

- Stakeholder requirements.
- Architecture Repository data.
- Viewpoint catalogue and standards (e.g. ArchiMate)
- Architecture principles and guidelines

## Outputs

- Architecture views and models.
- Viewpoint catalogue.
- Validated and approved architecture views.

## Roles

- Business Architect, Application Architect, Data Architect, Technology Architect (R)
- Lead Enterprise Architect (A)

## Common Issues with an Immature Capability

- Miscommunication between stakeholders and unclear views on the current situation
- Overwhelming complexity of architecture views (i.e. too detailed) or lack of relevance of views
- Creating views takes a long time with manual effort of multiple people

## Pragmatic Approach – Essentials That Are Needed for Success

- Build a pragmatic but up-to-date repository of your enterprise architecture
- Focus on defining key views with clear purpose and identify the necessary data for them
- Integrate data from existing repositories (e.g. CMDB) and automate creation of key views
