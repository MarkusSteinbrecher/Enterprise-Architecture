# Capability 5: Target State Design

**Domain:** Architecture Development - Common

## Description

Define the desired future architecture state aligned with business strategy, goals, and guiding principles. This capability translates strategic intent into concrete architecture blueprints across all domains.

## Objectives

- Define a clear and achievable target architecture aligned with strategy
- Ensure the target state addresses current gaps and weaknesses
- Create architecture blueprints that guide implementation
- Balance ambition with feasibility and pragmatism

## Key Activities

- Strategic alignment and vision development
- Target state architecture design across domains
- Architecture principles definition and application
- Option analysis and trade-off evaluation
- Blueprint creation (business, application, data, technology)
- Target state validation with stakeholders
- Feasibility and impact assessment

## Inputs

- Business strategy and goals
- Current state assessment and gap analysis
- Architecture requirements
- Architecture principles and standards
- Industry trends and best practices
- Constraints (budget, timeline, technology)

## Outputs / Deliverables

- Target state architecture models and blueprints
- Architecture principles catalog
- Option analysis and trade-off documentation
- Architecture decision records (ADRs)
- Transition architecture definitions
- Gap analysis results

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | No formal target state; changes are reactive |
| Developing | Target state defined for individual projects |
| Defined | Enterprise-wide target state defined; aligned with strategy |
| Managed | Target state regularly reviewed and updated; used to guide decisions |
| Optimizing | Adaptive target state; continuously evolving with strategy and technology |

## Key Roles

- Chief Architect
- Enterprise Architect
- Domain Architects
- Solution Architect
- Business strategists and planners

## Tools & Techniques

- Architecture modelling tools
- Scenario planning
- Trade-off analysis (ATAM, cost-benefit)
- Reference architectures
- Architecture patterns and blueprints
- Strategic planning frameworks

## KPIs / Metrics

- Strategic alignment score of target architecture
- Percentage of initiatives aligned with target state
- Number of architecture decisions documented
- Stakeholder approval rate for target designs
- Gap closure rate over time

## Relationships

- **Current State Assessment** - Gap between current and target drives design
- **Requirements Management** - Requirements constrain and shape the target
- **Roadmap Development** - Target state is the destination for the roadmap
- **Standards & Reference Architectures** - Standards and patterns inform target design
- **Architecture Review & Approval** - Target state reviewed and approved by governance
