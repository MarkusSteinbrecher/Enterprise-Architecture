# Capability 27: Architecture Review & Approval

**Domain:** Governance

## Description

Review and approve architecture deliverables, designs, and decisions to ensure they align with standards, principles, and strategic direction. This capability provides quality assurance and governance over architecture outputs.

## Objectives

- Ensure architecture deliverables meet quality standards
- Validate alignment with architecture principles and standards
- Provide constructive feedback to improve architecture designs
- Maintain consistency across architecture decisions

## Key Activities

- Architecture review process definition
- Architecture Review Board (ARB) facilitation
- Design review execution (solution, integration, security)
- Architecture compliance checking
- Review criteria and checklist maintenance
- Review findings tracking and resolution
- Lightweight review mechanisms (architecture clinics, office hours)
- Review process improvement

## Inputs

- Architecture deliverables for review
- Architecture standards and principles
- Review criteria and checklists
- Previous review findings
- Architecture requirements and constraints
- Risk assessments

## Outputs / Deliverables

- Architecture review reports
- Review findings and recommendations
- Approval / conditional approval / rejection decisions
- Architecture waivers (where applicable)
- Review metrics and trend reports
- Updated review criteria and checklists

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | No formal architecture reviews; quality is inconsistent |
| Developing | Ad-hoc reviews for major projects; basic checklists |
| Defined | Formal review process; ARB established; consistent criteria |
| Managed | Reviews integrated into delivery lifecycle; findings tracked to resolution |
| Optimizing | Automated compliance checking; reviews add clear value; continuous improvement |

## Key Roles

- Chief Architect (ARB chair)
- Enterprise Architect
- Domain Architects (reviewers)
- Solution Architect (submitter)
- Architecture Review Board members

## Tools & Techniques

- Review workflow tools
- Architecture fitness functions
- Compliance checking tools
- Review templates and checklists
- Architecture decision records
- Lightweight review formats (ADR reviews, design clinics)

## KPIs / Metrics

- Number of reviews conducted per period
- Review cycle time (submission to decision)
- Review finding severity distribution
- Finding resolution rate and time
- Stakeholder satisfaction with review process
- Compliance rate (pass/fail ratio)

## Relationships

- **Standards & Reference Architectures** - Standards are the basis for review criteria
- **Compliance Management** - Reviews support compliance assurance
- **Target State Design** - Target state designs are reviewed
- **Roadmap Development** - Roadmap items reviewed for feasibility
- **Architecture Communication** - Review outcomes communicated
