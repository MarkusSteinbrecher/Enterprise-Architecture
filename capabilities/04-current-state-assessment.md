# Capability 4: Current State Assessment

**Domain:** Architecture Development - Common

## Description

Analyze and document the existing architecture landscape to establish a baseline. This includes understanding the current business, application, data, and technology architectures, their strengths, weaknesses, and alignment to strategy.

## Objectives

- Establish an accurate and comprehensive baseline of the current architecture
- Identify strengths, weaknesses, risks, and technical debt
- Understand dependencies and relationships across architecture layers
- Provide a foundation for gap analysis and target state planning

## Key Activities

- Discovery and inventory of current architecture components
- Assessment of current state against business objectives
- Technical debt identification and classification
- Dependency and impact analysis
- Baseline documentation and modelling
- SWOT analysis of current architecture
- Health checks and fitness assessments

## Inputs

- Existing documentation and system inventories
- Application and technology portfolios
- Business process documentation
- Stakeholder interviews and surveys
- Monitoring and performance data

## Outputs / Deliverables

- Current state architecture models (baseline)
- Architecture inventory / catalog
- Technical debt register
- Dependency maps
- SWOT / fitness assessment reports
- Gap analysis inputs

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | No formal baseline; knowledge is tribal |
| Developing | Partial inventory maintained; assessments done per project |
| Defined | Comprehensive baseline documented; regular assessment cycles |
| Managed | Automated discovery and continuous baseline maintenance |
| Optimizing | Real-time architecture observability; predictive health analysis |

## Key Roles

- Enterprise Architect
- Domain Architects (Business, Application, Data, Technology)
- Solution Architect
- IT Operations / Infrastructure teams

## Tools & Techniques

- Discovery and inventory tools
- Architecture modelling tools
- Configuration management databases (CMDB)
- Application portfolio analysis tools
- Assessment frameworks and checklists

## KPIs / Metrics

- Percentage of architecture landscape documented
- Baseline currency (last update date)
- Number of identified technical debt items
- Assessment coverage across domains
- Accuracy of baseline documentation

## Relationships

- **Architecture Modelling** - Baseline captured through models
- **Target State Design** - Current state is the starting point for target design
- **Roadmap Development** - Gap analysis between current and target drives roadmap
- **Application Portfolio Management** - Application portfolio is part of current state
- **Risk Management** - Current state risks feed into risk management
