# Capability 21: Architecture Tooling

**Domain:** Architecture Repository

## Description

Select, deploy, configure, and manage tools that support architecture activities including modelling, repository management, portfolio analysis, and collaboration. The right tools amplify architecture effectiveness.

## Objectives

- Provide effective tools for all architecture activities
- Ensure tool integration and data consistency
- Maximize tool adoption and utilization
- Manage tool costs and licensing

## Key Activities

- Architecture tool landscape assessment
- Tool selection and evaluation
- Tool deployment and configuration
- Tool integration (connecting modelling, repository, portfolio tools)
- User training and adoption support
- Tool administration and maintenance
- License management
- Tool roadmap planning

## Inputs

- Architecture process requirements
- User needs and feedback
- Tool market analysis
- Budget constraints
- Integration requirements
- Security and compliance requirements

## Outputs / Deliverables

- Architecture tool landscape map
- Tool evaluation and selection reports
- Tool configuration and integration documentation
- Training materials and guides
- Tool usage reports
- Tool roadmap

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | General-purpose tools (PowerPoint, Visio); no dedicated architecture tools |
| Developing | Basic architecture modelling tool adopted; limited integration |
| Defined | Integrated architecture tool suite; standard configurations |
| Managed | Tools fully integrated; high adoption; automated workflows |
| Optimizing | AI-augmented architecture tools; seamless toolchain; continuous improvement |

## Key Roles

- Architecture Tooling Administrator
- Enterprise Architect
- Architecture Manager
- IT Procurement
- Vendor management

## Tools & Techniques

- Architecture modelling tools (Archi, Sparx EA, LeanIX, Mega, Ardoq)
- Repository and wiki tools (Confluence, SharePoint)
- Portfolio management tools
- Diagramming tools (draw.io, Miro, Lucidchart)
- Collaboration tools
- Tool evaluation frameworks (weighted scoring, proof of concept)

## KPIs / Metrics

- Tool adoption rate (active users / licensed users)
- User satisfaction with tools
- Tool integration coverage
- Tool cost per architect
- Time saved through tool automation
- Number of architecture artifacts managed in tools

## Relationships

- **Architecture Modelling** - Modelling depends on appropriate tools
- **Architecture Knowledge Management** - Repository tools store knowledge
- **Architecture Practice Management** - Tools enable the practice
- **Platform Architecture** - Architecture tools may be part of platform offerings
- **Architecture Communication** - Tools support architecture communication
