# Capability 9: Organization & Process Modelling

**Domain:** Business Architecture

## Purpose

To model organizational structures, roles, responsibilities, and business processes, understanding how work gets done and how the organization is structured to deliver on its capabilities and value streams.

## Scope

- Organizational structure and role modelling
- Business process modelling and documentation
- Process-to-capability and process-to-value stream mapping
- Process performance analysis and optimization

## Key Activities

- Model organizational structures, units, and reporting lines
- Define roles, responsibilities, and competency requirements
- Document business processes using standard notation (BPMN)
- Create process hierarchies and decompositions
- Map processes to business capabilities and value streams
- Analyze process performance (efficiency, effectiveness, compliance)
- Identify automation and optimization opportunities
- Define process governance frameworks
- Manage process standardization and harmonization across the enterprise
- Assess organizational design alignment with target architecture

## Inputs

- Business capability model (Cap 7)
- Value stream maps (Cap 8)
- Current organizational charts and role descriptions
- Existing process documentation
- Regulatory and compliance requirements
- Performance data and benchmarks

## Outputs / Deliverables

- Organizational models (structure, roles, reporting)
- Business process models (BPMN diagrams)
- Process catalog / repository
- Process-to-capability mapping
- Process-to-value stream mapping
- Process performance reports
- Automation opportunity assessments
- Organizational design recommendations

## Roles & Responsibilities

| Role | Responsibility |
|------|---------------|
| Business Architect | Leads organizational and process modelling |
| Process Analyst / Engineer | Details and optimizes process models |
| Process Owners | Own and validate their processes |
| HR / Organization Design | Supports organizational modelling |

## Maturity Levels

| Level | Name | Description |
|-------|------|-------------|
| 1 | Initial | Processes undocumented or documented inconsistently; organizational structures informal |
| 2 | Repeatable | Key processes documented; basic organizational models exist |
| 3 | Defined | Process architecture established; standard notation (BPMN) used; process repository maintained |
| 4 | Managed | Processes measured, governed, and actively improved; organizational alignment assessed |
| 5 | Optimizing | Intelligent process automation; continuous process optimization; organization design driven by architecture |

## Key Metrics / KPIs

- Process documentation coverage (% of key processes documented)
- Process standardization rate
- Process cycle time and efficiency
- Automation rate
- Organizational alignment score

## Dependencies

- **Business Capability Modelling** (Cap 7) - Processes realize business capabilities
- **Value Stream Mapping** (Cap 8) - Processes execute within value streams
- **Application Portfolio Management** (Cap 10) - Applications automate processes
- **Application Integration & API Management** (Cap 11) - Integration supports cross-process data flows
- **Architecture Compliance & Assurance** (Cap 28) - Processes must comply with regulations
