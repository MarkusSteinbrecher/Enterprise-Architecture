# Capability 20: Standards & Reference Architectures

**Domain:** Architecture Repository

## Description

Define, maintain, and govern architecture standards, patterns, principles, and reference architectures. These provide guardrails and reusable blueprints that guide consistent architecture decisions across the enterprise.

## Objectives

- Establish clear architecture standards and principles
- Provide reusable reference architectures and patterns
- Reduce architecture decision-making time and inconsistency
- Enable governance through well-defined standards

## Key Activities

- Architecture principles definition and maintenance
- Technology standards definition (approved, emerging, retiring)
- Reference architecture development
- Pattern catalog creation and management
- Standards lifecycle management
- Exception and waiver process management
- Standards communication and training
- Industry standard adoption and alignment

## Inputs

- Enterprise strategy and goals
- Industry standards and best practices
- Technology landscape and trends
- Architecture review feedback
- Vendor and product assessments
- Regulatory requirements

## Outputs / Deliverables

- Architecture principles catalog
- Technology standards (Technology Radar)
- Reference architectures
- Pattern catalog (design, integration, security)
- Standards compliance checklists
- Exception / waiver register
- Standards roadmap

## Maturity Indicators

| Level | Description |
|-------|-------------|
| Initial | No formal standards; decisions are inconsistent |
| Developing | Basic technology standards defined; limited adoption |
| Defined | Comprehensive standards and reference architectures; governance in place |
| Managed | Standards actively enforced; regular review cycles; exceptions managed |
| Optimizing | Living standards; community-driven updates; automated compliance checking |

## Key Roles

- Enterprise Architect
- Chief Architect
- Domain Architects
- Architecture Review Board
- Technology strategists

## Tools & Techniques

- Technology Radar (ThoughtWorks model)
- Architecture decision records
- Reference architecture templates
- Pattern description formats
- Standards governance frameworks
- Architecture fitness functions

## KPIs / Metrics

- Standards adoption rate
- Number of exceptions / waivers
- Reference architecture reuse rate
- Standards review cycle time
- Compliance rate in architecture reviews
- Standards coverage across domains

## Relationships

- **Target State Design** - Standards guide target architecture
- **Application Design** - Standards constrain application design choices
- **Architecture Review & Approval** - Standards are the basis for reviews
- **Compliance Management** - Standards compliance tracked and enforced
- **Architecture Knowledge Management** - Standards stored in the repository
